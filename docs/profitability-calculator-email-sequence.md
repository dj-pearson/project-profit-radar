## Technical Implementation

The email sequence is triggered automatically when a lead is captured in the calculator.

**Database Flow**:
1. User enters email in calculator
2. Record created in `calculator_leads` table
3. Webhook fired to Resend API
4. User added to "Calculator Nurture" audience
5. Automation sequence begins

**Code Integration** (see `/src/lib/calculatorAnalytics.ts`):
```typescript
await trackEmailCapture(sessionId, email, companyName, phone);
```

This function:
1. Stores lead in Supabase
2. Triggers webhook to Resend
3. Begins email sequence

**Webhook Endpoint** (to be created in Supabase Edge Functions):
```
POST /functions/v1/trigger-calculator-nurture
{
  "email": "contractor@example.com",
  "first_name": "John",
  "project_type": "Kitchen Remodel",
  "profit_margin": 22.5,
  "profit_amount": 8500,
  "risk_level": "medium"
}
```

## Email Deliverability Best Practices

1. **Warm up domain**: Gradually increase email volume
2. **Monitor bounce rate**: Keep under 2%
3. **Honor unsubscribes**: Immediately remove from list
4. **Authenticate emails**: Set up SPF, DKIM, DMARC
5. **Avoid spam triggers**: Don't use ALL CAPS excessively, limit exclamation points
